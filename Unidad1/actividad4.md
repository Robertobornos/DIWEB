# Principios del diseño interactivo

## 1. Visibilidad
Los elementos importantes deben ser fáciles de ver para que el usuario sepa qué puede hacer.

**Ejemplo:**  
Un botón de Comprar grande y de color llamativo en una tienda online.

---

## 2. Retroalimentación (Feedback)
El sistema debe informar al usuario sobre lo que está pasando después de realizar una acción.

**Ejemplo:**  
Cuando envías un formulario y aparece el mensaje “Formulario enviado correctamente”.

---

## 3. Consistencia
Los elementos del diseño deben funcionar de forma similar en toda la interfaz para que el usuario no tenga que aprender cosas nuevas constantemente.

**Ejemplo:**  
Todos los botones de una web tienen el mismo estilo y comportamiento.

---

## 4. Simplicidad
La interfaz debe ser sencilla y evitar elementos innecesarios que puedan confundir al usuario.

**Ejemplo:**  
Una página de inicio con pocas opciones claras en lugar de muchos menús complicados.

---

## 5. Control del usuario
El usuario debe sentir que tiene el control de la aplicación y poder deshacer acciones si es necesario.

**Ejemplo:**  
Un botón de Deshacer después de borrar un archivo.

---

## 6. Accesibilidad
El diseño debe permitir que personas con diferentes capacidades puedan usar la interfaz.

**Ejemplo:**  
Textos con buen contraste para que puedan ser leídos fácilmente por personas con problemas de visión.

---

## 7. Prevención de errores
El sistema debe ayudar a evitar que el usuario cometa errores.

**Ejemplo:**  
Un formulario que avisa si un correo electrónico está mal escrito antes de enviarlo.

---

## 8. Flexibilidad
La interfaz debe adaptarse a diferentes usuarios y formas de uso.

**Ejemplo:**  
Una web que funciona correctamente tanto en ordenador como en móvil.
