# ¿Qué es un CMS?

Un **CMS (Content Management System o Sistema de Gestión de Contenidos)** es una herramienta que permite crear y administrar páginas web sin tener que programar directamente.  
A través de un panel de administración, los usuarios pueden añadir, modificar o eliminar contenidos como textos, imágenes o vídeos de manera sencilla.

Normalmente un CMS funciona con dos partes principales:

- **Backend:** es la zona de administración donde se gestiona el contenido del sitio.
- **Frontend:** es la parte visible de la web que ven los visitantes, donde el contenido se muestra utilizando plantillas o temas.

Gracias a este tipo de sistemas es posible gestionar distintos tipos de páginas web, como blogs, páginas corporativas o tiendas online, sin necesidad de conocimientos técnicos avanzados.

---

# Ejemplos de uso de un CMS

## Caso en el que sí usaría un CMS
**Portal web de un ayuntamiento**

Un ayuntamiento que necesita publicar noticias, información de servicios, eventos o avisos para los ciudadanos podría utilizar un CMS como **WordPress** o **Joomla**.  
Este tipo de herramienta permite que diferentes trabajadores del ayuntamiento puedan actualizar los contenidos de la página sin depender constantemente de programadores.

Además, muchos CMS permiten crear **roles de usuario**, lo que facilita que cada departamento gestione su propia sección dentro del portal.

---

## Caso en el que no usaría un CMS
**Aplicación web con análisis de datos en tiempo real**

Si una empresa tecnológica necesita desarrollar una plataforma que procese datos en tiempo real y muestre resultados mediante gráficos o paneles personalizados, un CMS no sería la mejor opción.

Esto se debe a que los CMS están pensados principalmente para **gestionar contenidos** como artículos, imágenes o páginas informativas, pero no para ejecutar procesos complejos o cálculos continuos sobre grandes cantidades de datos.

En este caso sería más adecuado desarrollar la aplicación utilizando **frameworks de programación específicos**.

# Ejemplos de CMS y comparación

| CMS | Enfoque principal | Facilidad de uso | Nivel de personalización | Uso más habitual |
|----|----|----|----|----|
| **WordPress + WooCommerce** | Plataforma versátil con opción de comercio electrónico | Alta | Muy alta | Blogs, páginas corporativas y tiendas pequeñas o medianas |
| **Shopify** | Plataforma especializada en comercio electrónico | Muy alta | Media | Tiendas online rápidas de crear |
| **PrestaShop** | CMS enfocado a eCommerce con mayor control técnico | Media | Alta | Tiendas online medianas o grandes |
| **Wix eCommerce** | Constructor visual con funciones de tienda online | Muy alta | Media | Pequeños negocios o emprendedores |

---

# CMS recomendado para crear una tienda online

Si el objetivo es crear **una tienda online de forma rápida y funcional**, una buena opción sería **Shopify**.  
Esto se debe a que está diseñado específicamente para comercio electrónico, incluye herramientas de pago, gestión de productos y plantillas listas para usar.

Otra alternativa muy utilizada es **WordPress con WooCommerce**, que ofrece una gran flexibilidad y muchas opciones de personalización gracias a sus plugins.
